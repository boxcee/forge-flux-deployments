**setup.md changes:**

1. **Section 2A (FluxCD secret configuration):** Replace the `forge variables set WEBHOOK_SECRET` instruction with:
   - Navigate to Jira Settings (gear icon) > Apps > GitOps Deployments
   - Enter your HMAC shared secret in the "FluxCD HMAC Secret" field (min 8 characters)
   - Click Save -- success confirmation appears inline
   - Use the same secret value when creating the Kubernetes secret for your FluxCD notification provider

2. **Section 2C (FluxCD provider config):** Replace the vague "Replace `<webtrigger-url>` with the URL provided by your Jira app configuration" with: "Copy the FluxCD webhook URL displayed on the admin settings page."

3. **Section 3A (ArgoCD token configuration):** Replace `forge variables set ARGOCD_WEBHOOK_TOKEN` with admin UI instructions (same pattern as 2A but for the "ArgoCD Bearer Token" field).

4. **Section 3B (ArgoCD webhook service config):** Update webtrigger URL reference to say "Copy the ArgoCD webhook URL from the admin settings page."

5. **Section 2F and 3F (Verification response codes):** Add 503 to the response code lists:
   - `503` -- Webhook secret not configured. Navigate to the admin settings page to set your secret.

6. **Env var fallback note:** Add a brief admonition/note (just 1-2 lines) after the admin UI instructions in sections 2A and 3A: "Developers can alternatively use `forge variables set WEBHOOK_SECRET` / `ARGOCD_WEBHOOK_TOKEN` for CLI-based configuration. The admin UI takes priority when both are set."

7. **Verify sections:** Update references from `forge variables list --environment production` to checking the admin settings page.

**troubleshooting.md changes:**

1. **Add new entry: "503 -- Webhook secret not configured":**
   - Symptom: Webhook returns 503 with body "Webhook secret not configured. Configure via app admin page."
   - Cause: No secret has been saved in the admin settings page, and no env var fallback is set.
   - Fix: Navigate to Jira Settings > Apps > GitOps Deployments and enter your webhook secret.

2. **Add new entry: "Re-consent prompt after upgrading to v1.1":**
   - Symptom: Admin page shows error or KVS operations fail after app upgrade.
   - Cause: The v1.1 update adds `storage:app` scope, triggering a Forge major version bump that requires admin re-consent.
   - Fix: A Jira site admin must approve the new permissions. Existing webhooks continue working via env var fallback during the re-consent gap.

3. **Add new entry: "Admin page not found":**
   - Symptom: Cannot find the app's settings page.
   - Cause: Jira admin pages are under Settings > Apps (left sidebar), NOT in Manage Apps or the app overview page.
   - Fix: Navigate to Jira Settings (gear icon) > Apps > look for "GitOps Deployments" in the left sidebar. URL pattern: `/jira/settings/apps/{appId}/{envId}`.

4. **Update existing 401 entries:** Change references from `forge variables list --environment production` and `forge variables set` to checking/updating via the admin settings page. Keep env var as secondary mention.

5. **Update "Common mistakes quick reference" table at the bottom:** Replace `forge variables set` references with admin page references.
