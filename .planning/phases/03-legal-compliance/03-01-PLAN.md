<task type="auto">
  <name>Task 1: Enhance privacy policy with missing Marketplace-required sections</name>
  <files>docs/privacy-policy.md</files>
  <action>
Update docs/privacy-policy.md to add all missing sections identified in the gap analysis. Keep the existing sections (1-5) and enhance/extend them:

1. **Section 1 (Information We Collect)** -- expand "deployment metadata" to explicitly list: application names, namespaces, Helm chart versions, Git commit SHAs, Jira issue keys, environment names, deployment URLs. Add a clear statement that none of these constitute PII.

2. **Section 4 (Security)** -- add mention of HMAC signature verification for webhook authentication.

3. **Add Section 6: Data Deletion and Retention** -- State explicitly that no data is stored, so no deletion is necessary. Cover: on-uninstall behavior (no residual data, deployment records in Jira remain governed by Atlassian), deletion requests (no app-side data to delete, direct users to Atlassian for Jira records). Use the template from 03-RESEARCH.md "Code Examples" section.

4. **Add Section 7: Data Subject Rights** -- Since no PII is collected or stored, there is no personal data to access, modify, or delete. Direct users to Atlassian for data within Jira.

5. **Add Section 8: Changes to This Policy** -- Standard clause: policy may be updated, changes posted to this page, "Last Updated" date reflects changes.

6. **Add "Effective Date: March 10, 2026"** below the Last Updated line.

7. **Add brief GDPR/CCPA "not applicable" statements** in a new Section 9 (International Data): Since no PII is collected or stored, GDPR data subject rights and CCPA consumer rights do not apply to the App. Users should refer to Atlassian's privacy policy for data held within Jira.

Keep the same tone, Markdown formatting, and heading style as existing sections. Do NOT change the Jekyll frontmatter (title, nav_order, description).
  </action>
  <verify>
    <automated>grep -c "Data Deletion\|Data Subject Rights\|Changes to This Policy\|Effective Date\|HMAC" docs/privacy-policy.md | xargs test 5 -le</automated>
  </verify>
  <done>Privacy policy contains all 9 sections: Information We Collect (with explicit data list), How Data is Processed, Third-Party Services, Security (with HMAC), Contact, Data Deletion and Retention, Data Subject Rights, Changes to This Policy, International Data. Effective date present.</done>
</task>

<task type="auto">
  <name>Task 2: Enhance terms of service with missing Marketplace-required sections</name>
  <files>docs/terms-of-service.md</files>
  <action>
Update docs/terms-of-service.md to add all missing sections identified in the gap analysis. Keep existing sections (1-5) and add:

1. **Add Section 6: Termination** -- Agreement effective until terminated. User may terminate by uninstalling the App. Provider may terminate or suspend access at any time without prior notice. Upon termination, all rights granted cease immediately. Sections 2 (Disclaimer of Warranty), 3 (Limitation of Liability), and this section survive termination. Use the template from 03-RESEARCH.md.

2. **Add Section 7: Acceptable Use** -- Prohibited uses: reverse engineering, decompiling, disassembling; unlawful purposes; circumventing security mechanisms (including HMAC verification); impairing the Atlassian platform. Reference Atlassian Acceptable Use Policy (already linked in Section 5). Use the template from 03-RESEARCH.md.

3. **Add Section 8: Modifications to Terms** -- Right to update terms at any time. Changes posted to this page with updated "Last Updated" date. Continued use after changes constitutes acceptance.

4. **Add Section 9: Governing Law** -- Keep it neutral/brief: "These terms shall be governed by and construed in accordance with applicable law, without regard to conflict of law provisions." Do NOT specify a jurisdiction (research recommended omitting or keeping neutral for individual developer projects).

5. **Add "Effective Date: March 10, 2026"** below the Last Updated line.

Keep the same tone (formal, concise), Markdown formatting, and heading style. Do NOT change the Jekyll frontmatter.
  </action>
