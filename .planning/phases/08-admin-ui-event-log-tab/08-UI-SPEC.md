# Phase 8 — UI Design Contract

> Visual and interaction contract for the Event Log tab in the Forge admin page. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Forge UI Kit) |
| Preset | not applicable |
| Component library | @forge/react (Atlassian UI Kit, wraps @atlaskit) |
| Icon library | not applicable (Forge UI Kit has no icon imports; use Lozenge/Badge for visual indicators) |
| Font | Atlassian system font stack (controlled by Forge runtime, not configurable) |

**Constraint:** Forge UI Kit `render: native` mode. No custom HTML, no custom CSS, no `<style>` tags. All layout and styling via `@forge/react` components and `xcss` prop. The Atlassian Design System tokens govern all spacing, color, and typography.

---

## Spacing Scale

Forge UI Kit uses Atlassian Design Tokens. Declare only the token names used in this phase:

| Token | ADS Value | Usage |
|-------|-----------|-------|
| `space.100` | 8px | Inline gaps between Lozenge and text, compact padding |
| `space.200` | 16px | Default element spacing, tab panel padding |
| `space.300` | 24px | Stack spacing between sections (matches existing Settings page) |

Exceptions: none. The existing `src/frontend/index.jsx` uses `space.300` for top-level `Stack` spacing. This phase continues that convention.

---

## Typography

Forge UI Kit controls typography. Declare the component-level contract:

| Role | Component | Props | Usage |
|------|-----------|-------|-------|
| Page heading | `Heading` | `as="h2"` | "GitOps Deployments Configuration" (existing, unchanged) |
| Section heading | `Heading` | `as="h3"` | None needed (tabs provide section context) |
| Body text | `Text` | (default) | Table cell content, stats values, empty/error messages |
| Label text | `Text` | `weight="bold"` or `Strong` | Stats strip labels ("Accepted", "Failed", "Skipped") |
| Stat number | `Heading` | `as="h3"` | Stats strip count values |

## Color

Forge UI Kit colors are token-based. No hex values -- use component appearance props:

| Role | Mechanism | Usage |
|------|-----------|-------|
| Dominant (60%) | Default surface | Page background (Forge runtime controls this) |
| Secondary (30%) | `Box` with `xcss({ backgroundColor: 'color.background.neutral' })` | Stats strip cards |
| Accent (10%) | Lozenge `appearance` prop | Status code badges only (see below) |
| Destructive | Not used | No destructive actions in this phase |

### Status Code Badge Mapping (UI-05)

| Status Code | Lozenge Appearance | Label |
|-------------|-------------------|-------|
| 200 | `success` | "200 OK" |
| 204 | `default` | "204 Skipped" |
| 400 | `removed` | "400 Bad Request" |
| 401 | `removed` | "401 Unauthorized" |
| 502 | `removed` | "502 Bad Gateway" |
| 503 | `removed` | "503 Service Unavailable" |
| Other 4xx/5xx | `removed` | "{code}" |

`Lozenge` appearance values: `success` = green, `default` = grey, `removed` = red. These map directly to requirements UI-05.

## Component Inventory

### Layout Structure

```
App
  Stack (space="space.300")
    Heading as="h2" -> "GitOps Deployments Configuration"
    Tabs (onChange -> track activeTab)
      TabList
        Tab -> "Settings"
        Tab -> "Event Log"
      TabPanel -> [existing Settings content, unchanged]
      TabPanel -> EventLogPanel
```

### EventLogPanel

```
EventLogPanel
  Stack (space="space.300")
    SourceFilter (Select component)
    StatsStrip (Inline with 3 stat cards)
    EventTable (DynamicTable)
    LoadMoreButton (Button, conditional)
```

### Component Details

**1. Source Filter (UI-03)**

| Property | Value |
|----------|-------|
| Component | `Select` |
| Options | `[{ label: "All sources", value: "" }, { label: "Flux", value: "flux" }, { label: "Argo", value: "argocd" }]` |
| Default | `""` (All sources) |
| Behavior | onChange triggers re-fetch of both `getEventStats` and `getEventLog` (resets pagination) |

**2. Stats Strip (UI-02)**

| Property | Value |
|----------|-------|
| Layout | `Inline space="space.200" spread="fill"` |
| Cards | 3x `Box` with `xcss({ backgroundColor: 'color.background.neutral', padding: 'space.200', borderRadius: 'border.radius.100' })` |
| Card content | `Stack` with label `Text` ("Accepted" / "Failed" / "Skipped") + count `Heading as="h3"` |
| Data source | `getEventStats` resolver (returns `{ accepted, failed, skipped }`) |
| Period | "Last 24 hours" -- display as subtle `Text` above the strip |

**3. Event Table (UI-03, UI-04)**

| Property | Value |
|----------|-------|
| Component | `DynamicTable` |
| Page size | 25 rows (from `getEventLog` resolver) |
| Columns | See table below |

| Column | Key | Width | Content |
|--------|-----|-------|---------|
| Time | `timestamp` | 20% | ISO timestamp formatted as relative or short datetime |
| Source | `source` | 10% | "Flux" or "Argo" (capitalized from `flux`/`argocd`) |
| Status | `status_code` | 15% | `Lozenge` with appearance per mapping above |
| Release | `release_name` | 20% | Plain text, or "--" if null |
| Environment | `env` | 10% | Plain text, or "--" if null |
| Deployments | (computed) | 10% | `{accepted}A / {rejected}R` or "--" if all null |
| Error | `error` | 15% | Truncated to 50 chars with ellipsis, or "--" if null |

**DynamicTable props:**
- `isLoading`: true during fetch
- `emptyView`: empty state component (see Copywriting)
- `rows`: mapped from `getEventLog` response
- `head`: column definitions with `key`, `content`, `isSortable: false` (server-side ordering only)

**4. Load More Button (UI-04)**

| Property | Value |
|----------|-------|
| Component | `Button` |
| Appearance | `default` |
| Label | "Load more" |
| Visibility | Only shown when `hasMore === true` from `getEventLog` response |
| Behavior | Calls `getEventLog` with `beforeTimestamp` and `beforeId` from last row of current data, appends results |
| Loading state | `isDisabled` + `Spinner` while fetching next page |

## Interaction States

### Loading

| State | Behavior |
|-------|----------|
| Initial load | `Spinner` centered in EventLogPanel while both stats and events fetch |
| Filter change | Table shows `isLoading` overlay (DynamicTable built-in), stats strip shows previous values until new data arrives |
| Load more | "Load more" button disabled with spinner icon; existing rows remain visible |

### Error

| State | Behavior |
|-------|----------|
| Stats fetch fails | Stats strip shows "0" for all counts; `SectionMessage appearance="warning"` above table: "Could not load stats. Event log may still be available." |
| Events fetch fails | `SectionMessage appearance="error"` replaces table: "Failed to load event log. Try refreshing the page." |
| Partial failure | Stats and events are independent fetches; one can fail without affecting the other |

## Constraints and Notes

1. **No custom CSS.** Forge `render: native` does not support `<style>` or CSS files. All visual customization is via `xcss` and component props.
2. **No custom icons.** Forge UI Kit does not expose icon imports in native render mode. Use `Lozenge` for status indicators.
3. **Timestamp formatting.** Forge runtime includes no date formatting library. Use `new Date(ts).toLocaleString()` or simple string slicing. Do not add a dependency for this.
4. **Existing Settings content untouched.** The current form-based Settings UI moves into the first `TabPanel` with zero changes.
5. **Tab state is client-side only.** No URL routing in Forge admin pages. Default tab is "Settings" (index 0) to preserve existing UX. "Event Log" is index 1.
